# 规则与提示词迭代器

## 什么时候使用

当用户反复纠正 AI 系统、手动调整输出，或希望把对话历史转化为可复用指令时，使用本 Skill。

适合场景：

- 将审阅反馈转化为系统提示词规则。
- 在多次失败后更新某个 Skill。
- 从最终定稿中提取风格、结构或决策规则。

## 输入

需要收集：

- 来源材料：对话、审阅记录、diff、日志或最终采纳版本。
- 观察到的失败或重复模式。
- 人工纠正内容。
- 复用目标：提示词规则、Skill 段落、清单或策略。
- 适用范围：单项目、单领域或全局规则。

## 工作流程

1. 识别重复模式。
2. 区分局部偏好和通用规则。
3. 提取最小可复用指令。
4. 将规则归入一个层级：
   - L1 表现层
   - L2 应用层
   - L3 逻辑层
   - L4 底层协议
5. 增加证据和反例。
6. 生成草稿补丁，而不是立刻覆盖主规则。
7. 在合并前定义回归检查。

## 输出约定

返回：

```json
{
  "rule_id": "简短稳定的规则 ID",
  "layer": "L1_Presentation | L2_Application | L3_Logic | L4_Base_Protocol",
  "instruction": "可复用指令",
  "evidence": ["支持该规则的证据"],
  "limits": ["该规则不应适用的范围"],
  "regression_checks": ["合并前检查"],
  "patch_target": "需要更新的文件或 Skill 段落"
}
```

## Codex 使用说明

Codex 可在检查仓库后更新本地提示词、Skill 或文档文件，并通过正常代码编辑流程展示补丁意图。

## Claude 使用说明

Claude 应重点判断规则是否清晰、是否过度泛化，以及规则应归入表现层、应用层、逻辑层还是底层协议。

## 边界

- 不因一次纠错就生成全局规则。
- 不保存私人、敏感或专有示例。
- 未经明确审核，不更新 L3 或 L4 规则。
- 当干净规则足够时，不保留嘈杂对话原文。

## 示例提示词

```text
使用规则与提示词迭代器。

来源：我反复纠正助手，要求避免文学化隐喻，并使用专业技术写法。
复用目标：更新写作 Skill 的风格规则。
输出：包含证据和回归检查的规则草稿补丁。
```
