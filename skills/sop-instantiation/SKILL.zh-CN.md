# 标准作业程序实例化引擎

## 什么时候使用

当用户有目标但缺少执行路径时，使用本 Skill。它会把战略意图转化为可以执行、审核和迭代的标准作业程序。

适合场景：

- 规划发布、招聘、研究流程、内容流水线或运营例行工作。
- 将宽泛指令转化为分阶段工作。
- 将个人工作习惯整理为可复用操作手册。

## 输入

需要收集：

- 目标和截止时间。
- 可用人员、工具和数据。
- 必要交付物。
- 质量标准和失败条件。
- 已知约束和依赖关系。

## 工作流程

1. 明确目标，并定义完成标准。
2. 将目标拆分为阶段。
3. 为每个阶段定义输入、动作、输出、负责人和验收检查。
4. 标记可能分叉的决策点。
5. 增加风险控制和恢复步骤。
6. 将流程转化为清单或可排期计划。
7. 增加复盘环节，用于把经验压缩成未来规则。

## 输出约定

返回：

1. `目标`
2. `范围`
3. `里程碑计划`
4. `SOP 表格`
5. `决策点`
6. `风险控制`
7. `复盘与压缩记录`

## Codex 使用说明

只有当用户要求实施时，Codex 才应创建或更新本地任务文件、文档、脚本或模板。

## Claude 使用说明

Claude 应重点输出清晰的流程设计、角色定义和决策逻辑。

## 边界

- 不把不确定假设写成固定指令。
- 不跳过验证步骤。
- 未经用户确认，不分配真实人员。
- 未经明确要求，不创建外部日历、消息或任务。

## 示例提示词

```text
使用标准作业程序实例化引擎。

目标：准备一个中英双语 AI Skill 库的 GitHub 公开发布。
截止时间：本周。
输出：里程碑、清单、风险和最终发布审核步骤。
```
