# 个人方法论引擎

## 什么时候使用

当用户想要的是长期可运行的个人工作系统，而不是一次性提示词、文档或代码修改时，使用本 Skill。

适合场景：

- 将个人工作流转化为可复用 AI 操作系统。
- 组合计划、执行、审核和规则更新。
- 长期维护某个项目或个人专属 Skill 库。

## 输入

需要收集：

- 用户反复出现的目标和工作领域。
- 现有 Skill、规则、提示词、文档或项目约定。
- 被采纳与被拒绝的工作样例。
- 反馈历史和纠错模式。
- 风险边界：隐私、安全、不可逆操作和适用范围。

## 工作流程

1. 定义方法论适用范围。
2. 将现有 Skill 映射为解压、压缩或闭环角色。
3. 建立四层规则模型：
   - L1 表现层
   - L2 应用层
   - L3 逻辑层
   - L4 底层协议
4. 定义运行闭环：
   - 任务输入
   - 解压
   - 执行
   - 结果采集
   - 压缩门
   - 回归检查
   - 合并或回滚
5. 创建规则更新标准。
6. 定义需要人工确认的节点。
7. 输出维护计划和版本规则。

## 输出约定

返回：

1. `方法论范围`
2. `Skill 地图`
3. `四层规则模型`
4. `智能体闭环`
5. `更新策略`
6. `人工确认节点`
7. `版本与维护计划`

## Codex 使用说明

Codex 可以维护该方法论在本地仓库中的形态：文件、Skill、catalog 条目、校验脚本和文档。它应通过仓库检查验证修改。

## Claude 使用说明

Claude 可以帮助设计概念系统、评估规则质量，并输出成熟的方法论文档。

## 边界

- 不把每一次纠错都自动升级为长期规则。
- 不将私人材料写入公开 Skill。
- 未经审核，不更新高层逻辑。
- 不创建无法解释规则来源的系统。

## 示例提示词

```text
使用个人方法论引擎。

目标：为开源 Skill 创建构建一套个人 AI 协作系统。
材料：现有 Skill、审阅纠错、仓库结构和发布清单。
输出：Skill 地图、四层规则、智能体闭环和维护策略。
```
