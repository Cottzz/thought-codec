# Personal Methodology Engine Example / 个人方法论引擎示例

## Scenario

A user wants to turn their AI collaboration habits into a reusable personal working system.

用户希望将自己的 AI 协作习惯转化为可复用的个人工作系统。

## Prompt

```text
Use the Personal Methodology Engine Skill.

Goal:
Build a personal AI collaboration system for creating and maintaining open-source skills.

Materials:
- Existing ThoughtCodec repository structure
- Five initial skills
- User preference: read, understand, plan, execute, review
- User constraint: pause when a problem appears, solve it, then continue

Output:
- Methodology scope
- Skill map
- Four-layer rule model
- Agentic loop
- Update policy
- Human approval points
- Versioning and maintenance plan
```

## Expected Output Shape

- Scope that explains what the methodology covers and excludes.
- Skill map separating decompression, compression, and loop skills.
- Four-layer rules for presentation, application, logic, and base protocol.
- Agentic loop from task intake to merge or rollback.
- Human approval points for public publishing, high-level rule changes, and sensitive data.
- Versioning plan for skill updates.

## Review Checks

- The system can explain why each rule exists.
- Corrections become draft patches before becoming permanent rules.
- Private examples are not stored in public skills.
- High-level logic changes require review.
