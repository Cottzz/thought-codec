# 架构级脚手架生成器

## 什么时候使用

当用户有一个业务想法、产品机制、工作流或系统需求，并希望将其解压为工程可执行结构时，使用本 Skill。

适合场景：

- 一个产品功能需要模块、实体、API 和实施阶段。
- 一套业务规则需要数据库模型和服务边界。
- 一个模糊系统想法需要先形成清晰架构，再进入编码。

## 输入

需要获取或推断：

- 业务目标和目标用户。
- 核心规则、状态和事件。
- 必要输入、输出和外部集成。
- 预算、技术栈、隐私、合规或部署目标等约束。
- 期望输出格式：Mermaid、文件树、API 规范、SQL Schema、任务计划，或全部输出。

## 工作流程

1. 用一句话重述系统目标。
2. 提取领域实体、状态流转和核心业务规则。
3. 强制区分计算与表现：
   - 计算：业务逻辑、数据状态、校验、API 行为。
   - 表现：UI 布局、报告形态、文案、图表或渲染视图。
4. 输出高层架构图。
5. 定义模块、职责和边界。
6. 定义数据模型和 API 契约。
7. 如需要编码，生成实施脚手架或仓库目录树。
8. 补充校验清单和开放问题。

## 输出约定

按以下顺序输出：

1. `系统摘要`
2. `假设`
3. `架构图`
4. `领域模型`
5. `模块边界`
6. `API 契约`
7. `实施脚手架`
8. `校验清单`
9. `开放问题`

## Codex 使用说明

Codex 应先检查现有仓库，再把架构转化为文件。实现时遵循本地模式，运行针对性检查，避免无关重构。

## Claude 使用说明

Claude 应重点保证推理质量、取舍说明、领域边界和清晰的架构文档。

## 边界

- 不编造没有依据的业务需求。
- 不把渲染逻辑混入核心计算。
- 架构边界稳定前，不直接生成生产代码。
- 示例中不得暴露密钥、私人数据或专有材料。

## 示例提示词

```text
使用架构级脚手架生成器。

目标：为宠物营养品店搭建会员积分系统。
规则：用户通过购买、评价和推荐获得积分。积分可兑换优惠券。会员等级影响积分倍率。
输出：Mermaid 架构图、数据模型、API 契约和实施脚手架。
```
