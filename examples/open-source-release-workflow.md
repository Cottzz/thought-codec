# Open-Source Release Workflow / 开源发布工作流

This example shows how the five ThoughtCodec skills can work together when preparing a public repository.

## 1. Decompress The Project Goal

Use `architecture-scaffolding` to convert the broad project idea into a repository structure, documentation map, catalog schema, and visual asset plan.

使用 `architecture-scaffolding` 将项目想法解压为仓库结构、文档地图、catalog schema 和视觉资产计划。

## 2. Instantiate The Release SOP

Use `sop-instantiation` to turn the release goal into milestones, checks, and a publish-ready workflow.

使用 `sop-instantiation` 将发布目标转化为里程碑、检查点和可执行发布流程。

## 3. Compress Feedback Into Rules

Use `rule-prompt-optimizer` after review to convert repeated corrections into durable style or structure rules.

审核后使用 `rule-prompt-optimizer` 将重复纠错压缩为长期风格或结构规则。

## 4. Extract Engineering Reuse

Use `code-abstraction-encapsulation` when scripts, templates, or validation checks start repeating.

当脚本、模板或校验规则开始重复时，使用 `code-abstraction-encapsulation` 提取工程复用。

## 5. Maintain The Loop

Use `personal-methodology-engine` to keep the library evolving through explicit review, regression checks, and versioned updates.

使用 `personal-methodology-engine` 通过显式审核、回归检查和版本化更新，让 Skill 库持续演进。
