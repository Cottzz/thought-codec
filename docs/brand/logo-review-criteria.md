# Logo Review Criteria

Use this scorecard to review each ThoughtCodec logo route.

| Criterion | Question | Score |
| --- | --- | --- |
| Silhouette | Does the mark remain recognizable in black and white? | 1-5 |
| Small Size | Does it still read at favicon size? | 1-5 |
| Concept | Does it fit compression, decompression, skill, or workflow without being too literal? | 1-5 |
| Distinction | Does it avoid common AI/code clichés? | 1-5 |
| System Fit | Can it extend to banner, social preview, docs, and icon usage? | 1-5 |
| Simplicity | Is the geometry disciplined and maintainable? | 1-5 |
| Reference Fit | Does it preserve the useful Gemini reference ideas without copying the image? | 1-5 |

## References Used

- [Microsoft Fluent 2 iconography](https://fluent2.microsoft.design/iconography) emphasizes recognizable, functional, easily understood icons, with scaling and complexity decisions tied to use case.
- [Apple Human Interface Guidelines app icons](https://developer.apple.com/design/human-interface-guidelines/app-icons) and [Material Design icon guidance](https://m3.material.io/styles/icons/overview) reinforce platform-level expectations around clarity, consistency, and system fit.

## Review Method

1. View the mark at 512px.
2. View or mentally reduce it to 64px, 32px, and 16px.
3. Check whether the black-only silhouette still works.
4. Compare it against the project meaning.
5. Reject any route that depends on explanation, decoration, or fragile detail.
