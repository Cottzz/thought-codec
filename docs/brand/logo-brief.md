# ThoughtCodec Logo Brief

## Brand Anchor

- Product name: ThoughtCodec
- Chinese name: 思维编解码
- Category: bilingual AI skill library and agent workflow system
- Audience: developers, AI power users, open-source contributors, operators building reusable AI workflows
- Core idea: compress repeated practice into reusable skills; decompress intent into execution

## Desired Personality

- Modern
- Precise
- Calm
- Engineering-oriented
- Open-source friendly
- Trustworthy

## Avoid

- Letter stacking such as obvious `T` + `C` monograms.
- Overloaded code symbols such as generic angle brackets as the main identity.
- Heavy gradients, glass effects, shadows, or decorative depth.
- Complex metaphors that need explanation before they are understood.
- Cute, playful, mascot-like, or consumer-social styling.
- Dense detail that collapses at 16px or 32px.

## Design Criteria

The logo direction should pass these checks before becoming the final repository identity:

1. **Silhouette:** recognizable as a simple black shape.
2. **Small-size legibility:** clear at 16px, 32px, and 64px.
3. **Semantic fit:** suggests codec, transformation, modular skill, or structured flow without becoming literal clip art.
4. **Distinctiveness:** not a generic code bracket, chat bubble, sparkle, brain, or hexagon mark.
5. **System fit:** works as app icon, favicon, README mark, GitHub social preview mark, and monochrome stamp.
6. **Color resilience:** works in single-color, light-background, and dark-background versions.
7. **SVG quality:** minimal paths, no unnecessary filters, clean geometry, maintainable structure.

## Current Status

The previous mark is rejected as a final direction. It relied too much on literal coding brackets, gradient styling, and explanatory detail. The next step is to evaluate multiple identity directions before selecting one final mark.

## Gemini Reference Translation

The accepted Gemini reference direction contributes these reusable ideas:

- A geometric processing core.
- Inputs converging toward the core.
- Outputs branching outward from the core.
- Deep navy and cool cyan engineering palette.
- Flat, modern, open-source software feel.
- A clean `input -> core -> output` visual logic.

The project should not copy:

- The `METHODOX` name unless the project is intentionally renamed.
- Decorative poster backgrounds.
- Sparkle/star marks.
- Detailed infinity-tree geometry that collapses at small sizes.
- Glow, bevel, or rendered mockup effects.

## Current Working Direction

The current working route is **Processing Flow**:

- Left-side parallel inputs compress into a central geometric core.
- The core uses a hexagonal mesh to suggest methodology processing.
- Right-side branches decompress into multiple outputs.
- Colors are limited to deep navy, slate, ice blue, and cyan.
- The logo avoids text, gradients, 3D treatment, and decorative environment.
